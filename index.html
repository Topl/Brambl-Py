class brambl.Brambl.Brambl(params)¶

Each sub-module may be initialized in one of three ways

1.Providing a separetly initialized Request and KeyManager instance. Each of these instances may be initialized using the static methods Requests or KeyManager available in the BramblJS class.

2.Providing custom configuration parameters needed to create new instances of each sub-module with the specified parameters

3.Providing minimal inputs (i.e. calling Brambl with only a string constructor arguement). This will create new instances of the sub-modules with default parameters. KeyManager will create a new keyfile and Requests will target a locally running instance of Bifrost.

Parameters

  • params (dictionary) – A password string or dictionary containing an instance of Requests and KeyManager

  • params['KeyManager'] (KeyManager object) – KeyManager object (may be either an instance or config parameters)

  • params['KeyManager]['password'] (string) – The password used to encrpt the keyfile

  • params['KeyManager']['instance'] (KeyManager instance) – A previously initialized instance of KeyManager

  • params['KeyManager']['keyPath'] (string) – Path to a keyfile

  • params['KeyManager']['constansts'] (dictionary) – Parameters for encrypting the user’s keyfile

  • params['Requests'] (Requests object) – Request object (may be either an instance or config parameters)

  • params['Requests']['url'] (string) – The chain provider to send requests to

  • params['Requests']['apikey'] (string) – Api key for authorizing access to the chain provider.

Returns

Brambl object

Return type

instance of Brambl

async pollTx(txId, options={'interval': 3, 'maxFailedQueries': 10, 'timeout': 90})¶

A function to initiate polling of the chain provider for a specified transaction. This function begins by querying ‘getTransactionById’ which looks for confirmed transactions only. If the transaction is not confirmed, the mempool is checked using ‘getTransactionFromMemPool’ to ensure that the transaction is pending. The parameter ‘numFailedQueries’ specifies the number of consecutive failures (when resorting to querying the mempool) before ending the polling operation prematurely.

Parameters

  • txId (type description) – The unique transaction ID to look for

  • options (type description) – Optional parameters in dictionary to control the polling behavior

  • options['timeout'] (number) – The timeout (in seconds) before the polling operation is stopped

  • options['interval'] (number) – The interval (in seconds) between attempts

  • options['maxFailedQueries'] (number) – The maximum number of consecutive failures (to find the unconfirmed transaction) before ending the poll execution

Returns

polling request

Return type

JSON

brambl.modules.KeyManager.getMAC(derivedKey, ciphertext)¶

Calculate message authentication code from secret (derived) key and encrypted text. The MAC is the keccak-256 hash of the byte array formed by concatenating the second 16 bytes of the derived key with the ciphertext key’s contents.

  • ciphertext (bytes string) – ciphertext Text encrypted with secret key.
